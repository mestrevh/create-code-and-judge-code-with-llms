Título: Pô, que mão
Topicos do problema: obi; decisão; ordenação; 
Tempo limite de execução: 1 segundo(s)
Descrição: <p><img class="" src="https://www.thehuxley.com/api/v1/problems/image/501088d2b6628c7fd9938c777bf4d93dc0cbf3281.jpg"/><br/></p><p>Um novo jogo se tornou popular entre jovens de todas as idades recentemente: o <i>Pô, que mão</i>. Trata-se de um jogo onde uma mão captura criaturas raras e depois as força a lutarem umas contra as outras. Uma verdadeira barbárie. Ainda assim, o jogo se tornou bastante popular. As criaturas são chamadas de <i>pô-que-mãos</i>. No jogo, você pode dar doces para as <i>pô-que-mãos</i>, para que elas fiquem mais fortes e evoluam. Como há poucos doces, nem sempre é possível evoluir todas as <i>pô-que-mãos</i> que um jogador possui. Um jogador tem exatamente 3 <i>pô-que-mãos</i>. Cada um deles necessita de uma quantidade de doces para evoluir. Conhecendo-se a quantidade de doces disponíveis, escreva um programa para determinar qual o maior número de <i>pô-que-mãos</i> que podem evoluir.<br/></p>
Formato da entrada: <p><span style="color: rgb(12, 22, 58);">A entrada é composta por quatro linhas, cada uma contendo um inteiro. A primeira linha contém <b>N</b>, o número de doces disponíveis. A segunda linha contém <b>X</b>, o número de doces necessários para a primeira <i>pô-que-mão</i> evoluir. A próxima linha contém <b>Y</b>, o número de doces necessários para a segunda <i>pô-que-mão</i> evoluir. A última linha contém <b>Z</b>, o número de doces necessários para a terceira <i>pô-que-mão</i> evoluir.</span><br/></p><p><ul style="color: rgb(12, 22, 58);background-color: rgb(255, 255, 255);"><i></i><li>0 ≤ N ≤ 1000</li><li>1 ≤ X ≤ 1000</li><li>1 ≤ Y ≤ 1000</li><li>1 ≤ Z ≤ 1000</li></ul></p>
Formato da saída: <p><span style="color: rgb(12, 22, 58);">Seu programa deve produzir uma única linha, contendo um inteiro, o maior número possível de <i>pô-que-mãos</i> que podem evoluir.</span><br/></p>
