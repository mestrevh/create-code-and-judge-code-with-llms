Título: Um pouco de vida real
Topicos do problema: array; entrada formatada; substrings; repetição; função; string; 
Tempo limite de execução: 20 segundo(s)
Descrição: <p dir="ltr" id="docs-internal-guid-e6f9cd82-7fff-af04-7647-d230d5521582"><span style="color: #000000;background-color: transparent;">Bom, devia ter uma historinha aqui que fala como o fulano estava fazendo alguma coisa e pediu pro programador resolver o problema, e geralmente é alguma coisa matemática bem definida.</span></p><br/><p dir="ltr"><span style="color: #000000;background-color: transparent;">O negócio é o seguinte. Considere:</span></p><br/><p dir="ltr"><span style="color: #000000;background-color: transparent;">f(x) = x² + 5, para todo x </span><span style="color: #000000;background-color: transparent;"> ℕ</span></p><br/><p dir="ltr"><span style="color: #000000;background-color: transparent;">Bonitinho, certo? Uma função f bem definida para todo x que pertença ao conjunto dos números naturais. Para nós, programadores, matemáticos, engenheiros, isto funciona perfeitamente.</span></p><p dir="ltr"><span style="color: #000000;background-color: transparent;">O problema é quando você coloca a variável </span><span style="color: #000000;background-color: transparent;"><b>usuário</b> </span><span style="color: #000000;background-color: transparent;">na equação. Este digníssimo ignora os limites para qual a função é bem definida, e espera que se ele colocar um PEPINO na função que você a muito custo programou, ela vai funcionar e ele não vai ter que se preocupar com nada.</span></p><br/><p dir="ltr"><span style="color: #000000;background-color: transparent;">E acredite, você também não quer lidar com um usuário de software frustrado. Não quando o software é </span><b><span style="color: #000000;background-color: transparent;">seu</span></b><span style="color: #000000;background-color: transparent;">.</span></p><br/><p dir="ltr"><span style="color: #000000;background-color: transparent;"><span style="color: #000000;background-color: transparent;">    </span><span style="color: #000000;background-color: transparent;"></span>Então agora o desafio é o seguinte. Seu chefe chegou pra você no seu primeiro dia de trabalho (em um trabalho onde tudo é programado em C), dizendo que você precisaria popular um banco de dados para um cliente. Os dados desse banco chegaram hoje em planilhas Excel. Felizmente, alguns colegas de trabalho viram o que estava pra vir pro seu lado, e transformaram tudo em arquivos </span><a href="https://pt.wikipedia.org/wiki/Comma-separated_values"><span style="color: #1155cc;background-color: transparent;">CSV</span></a><span style="color: #000000;background-color: transparent;">.</span></p><br/><p dir="ltr"><span style="color: #000000;background-color: transparent;">    </span><span style="color: #000000;background-color: transparent;">O problema é que o </span><span style="color: #000000;background-color: transparent;">usuário</span><span style="color: #000000;background-color: transparent;"> não está nem aí com o jeito que ele deve escrever cada campo. Por exemplo, uma data de nascimento que deveria ser escrita como um ser humano normal escreveria - “13/13/2013” - pode estar escrita, <b>pelo </b></span><b><span style="color: #000000;background-color: transparent;">usuário</span></b><span style="color: #000000;background-color: transparent;"> da forma “13\13\2013”. Ou ainda,  “I3/I3/20I3” (a tecla ‘1’ do teclado dele estava quebrada). Ah, o horror.</span></p><br/><p dir="ltr"><span style="color: #000000;background-color: transparent;">    </span><span style="color: #000000;background-color: transparent;">Sua tarefa como programador é </span><span style="color: #000000;background-color: transparent;">consertar o arquivo CSV</span><span style="color: #000000;background-color: transparent;"> antes de inserir todos os dados no banco, para que nada exploda. Para facilitar, é garantido que sempre é possível consertar um dado inválido (ou seja, o  usuário não vai digitar “PEPINO” no campo “data_nascimento”)</span></p><br/><p dir="ltr"><span style="color: #000000;background-color: transparent;">    </span><span style="color: #000000;background-color: transparent;">Agora você me pergunta, quais as possíveis modificações que o usuário pode fazer no texto? Bom, boa sorte descobrindo isso. Não esqueça de me avisar depois, eu posso acabar aproveitando o seu código pra quando eu tiver que lidar com esse tipo de frustração novamente.</span></p><br/><p dir="ltr"><span style="color: #000000;background-color: transparent;">    O arquivo CSV desse banco tem oito colunas. Cada linha diz respeito a um usuário a ser cadastrado, e cada coluna contém uma informação do usuário na linha respectiva. O banco de dados é </span><span style="color: #000000;background-color: transparent;">muito rigoroso a inconsistências</span><span style="color: #000000;background-color: transparent;">, então se os dados de resposta de cada coluna não estiverem </span><span style="color: #000000;background-color: transparent;">perfeitamente formatados</span><span style="color: #000000;background-color: transparent;">, ele não realizará o cadastro. Todas as colunas são obrigatórias.</span></p><p dir="ltr"><span style="color: #000000;background-color: transparent;"><br/></span></p><p dir="ltr"><span style="color: #000000;background-color: transparent;"><br/></span></p><p dir="ltr"><span style="color: #000000;background-color: transparent;">Seguem as regras de cada campo:</span></p><pre><span style="color: #000000;background-color: transparent;">nome:<br/>    - apenas caracteres no intervalo ['A','z']<br/><br/>telefone_celular:<br/>    - Deve seguir o formato &#34;(xx)xxxxx-xxxx&#34;, onde x é um número de 0 a 9<br/><br/>cpf:<br/>    - 11 dígitos apenas<br/><br/>data_nasc:<br/>    - Deve seguir o formato &#34;dd/mm/yyyy&#34;<br/><br/>curso:<br/>    - Pode assumir apenas os valores &#34;EC&#34;, &#34;CC&#34;, &#34;SI&#34;<br/><br/>semestre:<br/>    - Deve seguir o formato &#34;yyyy.x&#34;, onde yyyy é o ano e x pode ser 1 ou 2, indicando o semestre<br/></span></pre><br/><p dir="ltr"><span style="color: #000000;background-color: transparent;">Exemplo de uma tabela </span><span style="color: #000000;background-color: transparent;"><b>perfeitamente formatada:</b></span></p><br/><pre><span style="color: #000000;background-color: transparent;">Nicolas Castro,(44)98233-5577,56732465401,24/07/1998,EC,2012.2</span></pre><br/><br/><p dir="ltr"><span style="color: #000000;background-color: transparent;">Nem o CSV de entrada como de saída possuem os valores de cabeçalho.</span></p><p dir="ltr"><span style="color: #000000;background-color: transparent;">O CSV de saída não deve ter espaços antes ou após as vírgulas.</span></p>
Formato da entrada: <p>Vida real não te dá formato de entrada.</p><p><br/></p><p>Good luck.<br/></p>
Formato da saída: <p>O mesmo CSV de entrada, mas corrigido segundo as regras acima, obviamente.<br/></p>
