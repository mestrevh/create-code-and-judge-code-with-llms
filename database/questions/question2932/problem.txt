Título: O Melhor Banco de Dados
Topicos do problema: estrutura de dados; Árvore; requer solução eficiente; 
Tempo limite de execução: 17 segundo(s)
Descrição: <p>Um Banco de Dados é uma coleção de dados inter-relacionados, representando informações sobre um domínio específico, ou seja, sempre que for possível agrupar informações que se relacionam e tratam de um mesmo assunto, posso dizer que tenho um banco de dados. Bancos de Dados são conhecidos por armazenar informações de forma eficiente e, geralmente, pouco custosas. Na maioria das vezes, ele é disposto na forma de uma árvore binária. Dado uma série de informações para serem armazenadas em um banco, você deverá fornecer algumas informações sobre os arquivos dispostos no seu banco. Sua missão é fazer um banco de dados que possua o melhor tipo de busca possível das informações pedidas!</p>
Formato da entrada: <p>A entrada consiste de algumas informações:</p><ol><li> O número <b>N</b> de arquivos que serão armazenados no banco <b>(1 &lt;= N &lt;= 40000)</b>;</li><li>Seguirão <b>N</b> linhas, cada uma contendo <b>um inteiro K</b> <b>(1 &lt;= K &lt;= 256)</b> que é uma chave identificadora do local em que se deseja armazenar a informação, <b>um</b> <b>inteiro X</b> <b>(0 &lt;= X &lt;= 40000)</b> que representará o id da informação a ser armazenada, <b>um inteiro</b> <b>Z </b>que representa o tamanho do nome da informação e <b>uma string S (tamanho de S &lt; 10)</b> que representa o nome da informação que será guardada;</li><li>Após as <b>N</b> linhas, será fornecido um <b>inteiro Y (Y &lt;= N) </b>que representa a quantidade de buscas a serem realizadas no banco de dados.</li><li>Seguirão <b>Y</b> linhas, cada uma com 4 inteiros: <b>ki</b>, que será a chave identificadora do grupo, <b>xi</b> que representará o grau de relevância da informação a ser armazenada, <b>zi</b> o tamanho do nome da informação <b>si</b> que é uma string, visando as buscas a serem feitas no banco. A busca será realizada no <b>ki-ésimo</b> grupo procurando-se pelo <b>si-ésimo </b>nome de informação.</li></ol>
Formato da saída: <p>Para cada uma das <b>yi </b>buscas, deverá ser impresso na tela a string &#34;Busca de número <b>i</b>:&#34;, onde <b>i</b> é o loop da busca que está sendo feita, seguido de todos os nós visitados na notação pré-ordem. Caso o nó não seja encontrado, deverá ser printado todos os nós, que foram visitados na busca e a string &#34;Elemento nao encontrado!&#34;.</p>
