Título: Alinhamento de Sequências em Espaço Linear
Topicos do problema: substrings; backtracking; recursão; dividir e conquistar; programação dinâmica; 
Tempo limite de execução: 1 segundo(s)
Descrição: <h1 style="color: rgb(0, 0, 0);">Alinhamento de Genomas em Espaço Linear</h1><h3 style="color: rgb(0, 0, 0);">Descrição</h3><p style="color: rgb(0, 0, 0);"><span><span class="citation-774"></span></span><span><span class="citation-774">Na Bioinformática, o alinhamento de sequências é crucial para identificar similaridades funcionais ou evolutivas entre organismos</span></span><span>.<span> </span></span><span><span class="citation-773"></span></span><span><span class="citation-773">O algoritmo clássico de Needleman-Wunsch resolve este problema garantindo o alinhamento ótimo global, mas utiliza uma matriz de tamanho<span> </span></span><span class="math-inline">$N \times M$</span></span><span>.</span></p><div class="source-inline-chip-container ng-star-inserted" style="color: rgb(0, 0, 0);"></div><div class="source-inline-chip-container ng-star-inserted" style="color: rgb(0, 0, 0);"></div><div class="source-inline-chip-container ng-star-inserted" style="color: rgb(0, 0, 0);"></div><p style="color: rgb(0, 0, 0);"><span>Para sequências curtas, isso é aceitável.<span> </span></span><span><span class="citation-772"></span></span><span><span class="citation-772">No entanto, ao comparar genomas reais com milhares de pares de bases, o consumo de memória quadrático<span> </span></span><span class="math-inline">$O(NM)$</span><span class="citation-772"><span> </span>torna-se proibitivo, estourando a memória RAM disponível</span></span><span>.</span></p><div class="source-inline-chip-container ng-star-inserted" style="color: rgb(0, 0, 0);"></div><div class="source-inline-chip-container ng-star-inserted" style="color: rgb(0, 0, 0);"></div><p style="color: rgb(0, 0, 0);"><span><span class="citation-771"></span></span><span><span class="citation-771">Sua tarefa é implementar uma solução eficiente que encontre o<span> </span></span><b><span class="citation-771">custo do alinhamento ótimo</span></b><span class="citation-771"><span> </span>e<span> </span></span><b><span class="citation-771">reconstrua as sequências alinhadas</span></b><span class="citation-771"><span> </span>utilizando apenas espaço linear<span> </span></span><span class="math-inline">$O(N+M)$</span></span><span>.</span></p><div class="source-inline-chip-container ng-star-inserted" style="color: rgb(0, 0, 0);"></div><div class="source-inline-chip-container ng-star-inserted" style="color: rgb(0, 0, 0);"></div><p style="color: rgb(0, 0, 0);"><span><span class="citation-770"></span></span><span><span class="citation-770">Para este problema, utilize o seguinte esquema de custos (penalidades) para minimização</span></span><span>:</span></p><div class="source-inline-chip-container ng-star-inserted" style="color: rgb(0, 0, 0);"></div><div class="source-inline-chip-container ng-star-inserted" style="color: rgb(0, 0, 0);"></div><div class="source-inline-chip-container ng-star-inserted" style="color: rgb(0, 0, 0);"></div><ul style="color: rgb(0, 0, 0);"><li><p><b>Match (bases iguais):</b><span> </span>Custo 0</p></li><li><p><b>Mismatch (bases diferentes):</b><span> </span>Custo 1</p></li><li><p><b>Gap (lacuna):</b><span> </span>Custo 2</p></li></ul><p style="color: rgb(0, 0, 0);"><span><span class="citation-769"></span></span><span><span class="citation-769">A estratégia recomendada é combinar a programação dinâmica com apenas duas colunas na memória (para calcular o score) com a técnica de Divisão e Conquista para recuperar o caminho (Algoritmo de Hirschberg)</span></span><span>.</span></p><p style="color: rgb(0, 0, 0);"><span><br/></span></p><p style="color: rgb(0, 0, 0);"><b class="">Para garantir que sua saída seja idêntica à do juiz, você deve seguir rigorosamente as seguintes regras de desempate:</b></p><ol start="1" style="color: rgb(0, 0, 0);"><li><p>Prioridade na Reconstrução (Casos Base):</p><p>Durante a fase de reconstrução do caminho (backtracking) nos casos base (quando a recursão atinge o tamanho limite), se houver empate entre os custos de transição, a prioridade de escolha deve ser nesta ordem exata:</p><ol start="1"><li><p><b class="">Diagonal</b><span class=""><span> </span>(Alinhar bases<span> </span></span><span class="math-inline">$X[i]$</span><span class=""><span> </span>e<span> </span></span><span class="math-inline">$Y[j]$</span><span class="">,</span><span class=""><span> </span>seja Match ou Mismatch).</span></p></li><li><p><b class="">Cima</b><span class=""><span> </span>(Alinhar<span> </span></span><span class="math-inline">$X[i]$</span><span class=""><span> </span>com um gap<span> </span></span><code class="" style="color: rgb(199, 37, 78);background-color: rgb(249, 242, 244);">-</code><span class="">).</span></p></li><li><p><b class="">Esquerda</b><span class=""><span> </span>(Alinhar um gap<span> </span></span><code class="" style="color: rgb(199, 37, 78);background-color: rgb(249, 242, 244);">-</code><span class=""><span> </span>com<span> </span></span><span class="math-inline">$Y[j]$</span><span class="">).</span></p></li></ol></li><li><p>Prioridade na Divisão (Corte de Hirschberg):</p><p>Ao calcular o ponto de corte na coluna central ($mid = N/2$), você deve somar os custos da esquerda ($L$) e da direita ($R$). Se houver múltiplos índices $i$ (linhas) que resultam na mesma soma mínima, escolha o menor índice $i$</p></li></ol><div class="source-inline-chip-container ng-star-inserted" style="color: rgb(0, 0, 0);"></div><div class="source-inline-chip-container ng-star-inserted" style="color: rgb(0, 0, 0);"></div><div class="source-inline-chip-container ng-star-inserted" style="color: rgb(0, 0, 0);"></div><h3 style="color: rgb(0, 0, 0);">Formato de Entrada</h3><p style="color: rgb(0, 0, 0);">A entrada consiste em duas linhas.</p><ul style="color: rgb(0, 0, 0);"><li><p>A primeira linha contém a sequência<span> </span><span class="math-inline">$X$</span>.</p></li><li><p>A segunda linha contém a sequência<span> </span><span class="math-inline">$Y$</span>.</p></li></ul><p style="color: rgb(0, 0, 0);">As sequências são formadas apenas pelos caracteres: 'A', 'C', 'G', 'T'.</p><p style="color: rgb(0, 0, 0);">O tamanho das sequências ($|X|$ e $|Y|$) pode chegar até $10^4$.</p><h3 style="color: rgb(0, 0, 0);">Formato de Saída</h3><p style="color: rgb(0, 0, 0);">A saída deve conter três linhas:</p><ol start="1" style="color: rgb(0, 0, 0);"><li><p>Um inteiro representando o<span> </span><b>Custo Total (Score)</b><span> </span>mínimo do alinhamento.</p></li><li><p>A sequência<span> </span><span class="math-inline">$X$</span><span> </span>alinhada (inserindo o caractere '-' onde houver gaps).</p></li><li><p>A sequência<span> </span><span class="math-inline">$Y$</span><span> </span>alinhada (inserindo o caractere '-' onde houver gaps).</p></li></ol><p style="color: rgb(0, 0, 0);"><i>Nota: Se houver múltiplos alinhamentos com o mesmo custo ótimo, qualquer um deles será aceito.</i></p><h3 style="color: rgb(0, 0, 0);">Exemplos</h3><p style="color: rgb(0, 0,
