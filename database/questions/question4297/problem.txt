Título: Salve o universo
Topicos do problema: requer solução eficiente; busca; 
Tempo limite de execução: 20 segundo(s)
Descrição: <p>A lenda urbana diz que, se você for à página inicial do Google e pesquisar por &#34;Google&#34;, o universo implodirá. Temos um segredo para compartilhar... É verdade! Por favor, não tente, nem conte a ninguém. Tudo bem, talvez não. Estamos apenas brincando.</p><p><span>O mesmo não é verdade para um universo muito distante. Nesse universo, se você pesquisar em qualquer mecanismo de pesquisa o nome desse mecanismo de pesquisa, o universo implode!</span></p><p><span>Para combater isso, as pessoas criaram uma solução interessante. Todas as consultas são agrupadas. Eles são passados para um sistema central que decide qual consulta vai para qual mecanismo de busca. O sistema central envia uma série de consultas para um mecanismo de busca e pode alternar para outro a qualquer momento. As consultas devem ser processadas na ordem em que são recebidas. O sistema central nunca deve enviar uma consulta a um mecanismo de pesquisa cujo nome corresponda à consulta. Para reduzir custos, o número de comutadores deve ser minimizado.</span></p><p><span>Sua tarefa é nos dizer quantas vezes o sistema central terá que alternar entre os mecanismos de pesquisa, supondo que o programemos de maneira otimizada.</span><br/></p>
Formato da entrada: <p>A primeira linha do arquivo de entrada contém o número de casos, <b>N</b>. <b>N</b> casos de teste a seguir.</p><p>Cada caso começa com o número <b>S</b> -- o número de motores de busca. As próximas <b>S</b> linhas contêm, cada uma, o nome de um mecanismo de pesquisa. Cada nome de mecanismo de pesquisa não tem mais de cem caracteres e contém apenas letras maiúsculas, letras minúsculas, espaços e números. Não haverá dois motores de busca com o mesmo nome.</p><p>A linha a seguir contém um número <b>Q</b> -- o número de consultas recebidas. Cada uma das próximas <b>Q</b> linhas conterá uma consulta. Cada consulta será o nome de um mecanismo de pesquisa no caso.<br/></p><p>0 &lt; <b style="color: rgb(32, 33, 36);">N</b> ≤ 20<br/></p><p>2 ≤ <b style="color: rgb(32, 33, 36);">S</b> ≤ 10<br style="color: rgb(32, 33, 36);"/>0 ≤ <b style="color: rgb(32, 33, 36);">Q</b> ≤ 100<br/></p>
Formato da saída: <p>Para cada caso de entrada, você deve produzir:</p><p>Caso # X: Y</p><p>onde <b>X</b> é o número do caso de teste e <b>Y</b> é o número de opções do mecanismo de pesquisa. Não conte a escolha inicial de um mecanismo de pesquisa como uma opção.<br/></p>
