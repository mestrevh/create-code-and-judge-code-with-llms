Título: Conversor de expressões
Topicos do problema: estrutura de dados; pilha; 
Descrição: <p style="color: rgb(0, 0, 0);background-color: rgb(255, 255, 255);"><span><font color="#000000"><font face="Helvetica Neue, Helvetica, Arial, sans-serif"><font size="2"><span><span><span>As linguagens de programação desempenham um papel fundamental no desenvolvimento de software e na construção de sistemas computacionais. Cada linguagem possui características específicas que a tornam mais adequada para determinados tipos de aplicação, nestas características podemos mencionar os paradigmas de programação. Muitos são bem conhecidos: Imperativo, Funcional, Orientado a objetos, ou uma combinação destes. Mas um não tão conhecido é o paradigma baseado em Pilha, onde a estrutura de controle e a manipulação de dados ocorrem por meio de uma pilha (stack), na qual valores são empilhados (push) e desempilhados (pop) conforme necessário para a execução das operações. Uma convenção neste tipo de paradigma é o uso da<span> </span></span></span></span></font></font></font></span><strong><span><font color="#000000"><font face="Helvetica Neue, Helvetica, Arial, sans-serif"><font size="2"><span><span><b><span>Notação Polonesa Reversa (RPN - Reverse Polish Notation)</span></b></span></span></font></font></font></span></strong><span><font color="#000000"><font face="Helvetica Neue, Helvetica, Arial, sans-serif"><font size="2"><span><span><span><span>, no qual em vez da notação infixa tradicional: (1 + 2 ) * 3; A RPN posiciona os operadores </span></span></span></span></font></font></font></span><strong><span><font color="#000000"><font face="Helvetica Neue, Helvetica, Arial, sans-serif"><font size="2"><span><span><b><span>após</span></b></span></span></font></font></font></span></strong><span><font color="#000000"><font face="Helvetica Neue, Helvetica, Arial, sans-serif"><font size="2"><span><span><span><span> os operandos, eliminando a necessidade de parênteses para definir a precedência das operações: 1 2 + 3 *.</span></span></span></span></font></font></font></span></p><p style="color: rgb(0, 0, 0);background-color: rgb(255, 255, 255);"><font color="#000000"><font face="Helvetica Neue, Helvetica, Arial, sans-serif"><font size="2">Sua missão, portanto, é converter uma expressão matemática tradicional para a RPN, de forma que esta possa ser utilizada em qualquer linguagem orientada a Pilha. </font></font></font></p><p style="color: rgb(0, 0, 0);background-color: rgb(255, 255, 255);"><font color="#000000"><font face="Helvetica Neue, Helvetica, Arial, sans-serif"><font size="2">- Os operadores que podem ser encontrados são: +, -, *, / e ^ (exponenciação).  </font></font></font></p><p style="color: rgb(0, 0, 0);background-color: rgb(255, 255, 255);"><font color="#000000"><font face="Helvetica Neue, Helvetica, Arial, sans-serif"><font size="2">- Os operandos são letras minúsculas: aa e abc, por exemplo, são operandos inválidos. </font></font></font></p>
Formato da entrada: <p style="color: rgb(0, 0, 0);background-color: rgb(255, 255, 255);"><i>t</i><span> </span>[ número de expressões &lt;=<span> </span><b>100</b>] </p><p style="color: rgb(0, 0, 0);background-color: rgb(255, 255, 255);"><i>expressão</i> [ 1 &lt;= tamanho &lt;=<span> </span><b>400</b>] </p><p style="color: rgb(0, 0, 0);background-color: rgb(255, 255, 255);">[ t-1 expressões ]</p><p style="color: rgb(0, 0, 0);background-color: rgb(255, 255, 255);"><br/></p><p style="color: rgb(0, 0, 0);background-color: rgb(255, 255, 255);">considere todas as entradas de teste em formato válido.</p>
Formato da saída: <p><span style="color: rgb(0, 0, 0);background-color: rgb(255, 255, 255);float: none;">As expressões na notação RPN, uma por linha</span><br/></p>
