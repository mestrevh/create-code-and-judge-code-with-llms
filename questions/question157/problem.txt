Título: Sacoleiro
Topicos do problema: estrutura de dados; grafo; array bidimensional; 
Descrição: <p>Seu amigo sacoleiro pediu sua ajuda num problema que ele est&aacute; enfrentando. Ele tem um mapa de cidades que ele j&aacute; conhece e que s&atilde;o interessantes para ele, al&eacute;m das rotas entre as mesmas. Ele pretende fazer uma viagem para comprar presentes para seu filho e para sua filha. O problema &eacute; que nem todos os presentes t&ecirc;m o mesmo pre&ccedil;o, alguns s&atilde;o obviamente mais caros que os outros, e ele n&atilde;o quer ser injusto dando presentes mais caros para um ou para outro. O objetivo &eacute; fazer com que diferen&ccedil;a entre a soma dos valores dos presentes seja a menor poss&iacute;vel (de prefer&ecirc;ncia que sejam iguais, naturalmente). H&aacute;, tamb&eacute;m, um limite de quanto ele pode gastar na viagem.</p>
<p>O sacoleiro tem um mapa com N cidades e as rotas que as ligam. Al&eacute;m disso, cada cidade pertence ao grupo A ou ao grupo B. No grupo A est&atilde;o as cidades em que h&aacute; presentes para o filho, enquanto que no grupo B est&atilde;o as cidades com presentes para a filha. Sempre que ele p&aacute;ra numa cidade ele pode comprar ou n&atilde;o o presente, mesmo que ele j&aacute; tenha estado l&aacute; antes, inclusive pode comprar mais de uma unidade do mesmo presente (enquanto tiver dinheiro dispon&iacute;vel, naturalmente). As cidades s&atilde;o numeradas de 0 a N - 1. O trajeto deve sempre come&ccedil;a na cidade 0. O tamanho do percurso n&atilde;o importa para o sacoleiro. O total dispon&iacute;vel de dinheiro para os presentes &eacute; T. O sacoleiro n&atilde;o pode terminar a viagem sem ter comprado pelo menos um presente para algum dos filhos.</p>
<p>Escreva um programa que, dadas N cidades, as rotas entre elas e os valores de presentes de cada cidade, retorne qual a diferen&ccedil;a m&iacute;nima poss&iacute;vel entre a soma dos presentes do grupo A e a soma dos presentes do grupo B.</p>
<p>&nbsp;</p>
Formato da entrada: <p>A entrada cont&eacute;m um &uacute;nico conjunto de testes. A primeira linha da entrada cont&eacute;m um inteiro N (2 &lt;= N &lt;= 30) que indica a quantidade de cidades. A segunda linha cont&eacute;m um inteiro T (10 &lt;= T &lt;= 100) que indica a quantidade de dinheiro que o sacoleiro tem para gastar. As N linhas seguintes cont&ecirc;m a descri&ccedil;&atilde;o cada cidade. Cada uma dessas linhas tem o formato X P C K V0 V1 ... VK-1, onde X &eacute; um inteiro que representa a cidade (numeradas de 0 a N - 1); P &eacute; um inteiro (1 &lt;= P &lt;= 10) que indica o valor do presente da cidade X; C &eacute; um caracter A ou B, indicando a que grupo a cidade X pertence; K &eacute; um inteiro (0 &lt;= K &lt; N) que indica quantas rotas saem da cidade X; e cada Vi &eacute; um inteiro indicando um dos poss&iacute;veis destinos a partir da cidade X. Note que as rotas n&atilde;o s&atilde;o bidirecionais. Uma cidade nunca ter&aacute; rota para ela mesma e pode-se assumir que i != j e Vi != Vj.</p>
Formato da saída: <p>Seu programa deve imprimir, na sa&iacute;da padr&atilde;o, uma &uacute;nica linha com um inteiro representando a menor diferen&ccedil;a poss&iacute;vel de valores entre os presentes comprados para o grupo A e para o grupo B.</p>
