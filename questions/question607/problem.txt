Título: Mantenha-se energizado
Topicos do problema: segment tree; binary indexed tree; maratona; requer solução eficiente; programação dinâmica; rmq; busca binária; 
Descrição: <p>A Incrível Companhia de Produção de Consoles (ICPC) está desenvolvendo seu mais novo console de videogame, o Super-Arcade Reloaded (SAR). O lançamento do SAR será acompanhado pelo lançamento de um jogo revolucionário, que estará disponível somente para os usuários do SAR. Este jogo será chamado de Aventuras do Capitão Mikado (ACM) e possui como destaque uma moeda virtual que pode ser comprada usando moeda real :-)</p><p>O ACM é um jogo muito simples e consiste de N fases numeradas de 1, 2, ..., N, onde a i-ésima fase requer exatamente Ei unidades de energia para ser completada. Isso significa que para completar aquela fase, a energia do usuário deve ser pelo menos Ei. Após completar o nível, a energia irá ser decrementada exatamente de Ei unidades. Para vencer o jogo, o usuário deve completar todas as fases em ordem crescente, começando no nível 1 e continuando até o nível N, sem nunca voltar para uma fase já completada.</p><p>Inicialmente o usuário começa sem nenhuma energia e, para ter alguma, ele precisa comprar pacotes de energia as lojas existentes dentro das fases do jogo. Existem M lojas. Cada loja vende pacotes de energia que possuem força S e custo C, que depende de cada loja. O usuário só pode comprar pacotes de energia nas lojas que existem na fase em que ele se encontra, e antes de iniciar a fase.</p><p>O efeito de comprar um pacote de com força S é que a energia do usuário se torna imediatamente S, independentemente de qual o valor de energia que usuário possuía antes.</p><p>Com o intuito de vender ainda mais, o ICPC pensou em uma promoção revolucionária: ele irá reembolsar o custo total do SAR para qualquer usuário que completar o jogo ACM usando a menor quantidade de moeda virtual possível. <br/></p><p>Você consegue encontrar qual é a menor quantidade de moeda virtual possível para completar o jogo?<br/></p>
Formato da entrada: <p></p><p>A primeira linha contém 2 inteiros N e M, representando respectivamente o número de fases e o número de lojas no jogo ( 1&lt;=N,M &lt;=10^5). A segunda linha contém N inteiros E1, E2, ..., EN, onde Ei é o nível de energia necessário para completar a i-ésima fase ( 1&lt;= Ei &lt;= 10^4, para i = 1, 2, ..., N).</p><p>Cada uma das próximas M linhas descreve uma loja com 3 inteiros L, S e C, representando respectivamente a fase onde a loja está localizada, a força e o custo para comprar o pacote de energia ( 1&lt;=L&lt;=N, 1&lt;=S&lt;=10^9 e 1&lt;= C&lt;= 10^4).<br/></p><br/>
Formato da saída: <p>Imprima uma linha com um inteiro representando a quantidade mínima de moeda virtual necessária para completar as N fases do jogo. Se for impossível completar todos os níveis, imprima o valor -1.<br/></p>
