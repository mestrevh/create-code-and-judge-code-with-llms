Rotas de Entrega

Tempo limite: 10 segundo(s)

A Compahia de Barcos da China(CBC) precisa de um software para ajudá-la a fazer cotações para os seus clientes. O custo de uma entrega de navio depende do tamanho da encomenda e de quantas aquavias são necessárias percorrer.

Uma aquavia é um caminho marítimo que conecta dois portos. Nem todos os pares de portos estão conectados por uma aquavia. Pode ser que para ir de um porto para o outro, seja necessário passar por várias aquavias.

A CBC opera em até 7000 portos diferentes. Cada porto é identificado por uma string. As aquavias sempre existem entre dois portos distintos. Todas as aquavias são bidirecionais.

O custo de uma entrega é igual ao tamanho da encomenda vezes o número de aquavias necessárias para ir de um porto de origem ao porto de destino vezes $100.


A entrada contém os portos e as aquavias. Para cada solicitação de cotação, existirá o tamanho da encomenda, o porto de origem e o porto de destino. O seu programa deve imprimir o menor custo possível para essa entrega, caso seja possível entregar no destino. Pois pode ser que não exista um caminho de aquavias entre a origem e o destino.

Formato da Entrada

A primeira linha conterá um inteiro de D ( 1<=D<=10) indicando o número de casos de teste. Cada caso de teste representa uma nova configuração.

A primeira linha de cada caso de teste contém 3 inteiros: M( 1<=M<=7000) , N  (1<=N<M)  e P (2<=P<=M).

  • M é um inteiro representando o número de portos.
  • N é um inteiro representando o número de aquavias.
  • P é um inteiro representando o número de cotações que serão realizadas.

A segunda linha do caso de teste contém M strings representando os nomes de cada porto. O tamanho máximo da string é 100 caracteres. As strings são separadas por espaços em branco.

Depois seguirão N linhas. Cada uma no formato XX YY, onde siginifica que existe uma aquavia entre o porto XX e o porto YY.

Depois seguirão P linhas. Cada linha consiste de um inteiro Z (1<=Z<=10) correspondendo ao tamanho da encomenda, seguido de um par XX YY, onde XX corresponde ao porto de origem e YY ao porto de destino.

Formato da Saída

A primeira linha de todas deve conter a string: "SHIPPING ROUTES OUTPUT"

Para cada caso de teste, deve existir uma seção enumerando o caso de teste que se seguirá. Depois seguirão P linhas consistindo do menor custo encontrado para a respectiva solicitação de cotação. Caso não seja possível realizar a entrega, você deve imprimri: "NO SHIPMENT POSSIBLE"

Na última linha da saída, você deve imprimir:  "END OF OUTPUT"
