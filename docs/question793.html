Roteamento

Tempo limite: 1 segundo(s)

Numa rede de dados, N nós (roteadores) são conectados entre si através de linhas comunicação. Um protocolo de roteamento é utilizado para determinar o `melhor' caminho (rota) que um pacote de dados deve seguir do nó emissor S até o nó receptor T da mensagem. Como o número de possíveis rotas entre dois nós cresce exponencialmente com a quantidade de pontos da rede, uma alternativa para simplificar a tarefa de roteamento consiste em eleger um nó da rede como "roteador designado" (Designeted Router - DR) e fazer com que um pacote de S para T sempre seja encaminhado via DR, ou seja, a mensagem segue SEMPRE a melhor rota de S até DR e, em seguida, de DR até T (mesmo que, no primeiro trecho S~>DR, ele já passe por T).

Cada linha (i,j) da rede possui tem uma largura de banda (bandwidth) B[i][j] medida em Kbps (Kilobits por segundo, 1Kbit = 2^10 bits). Para se calcular as melhores rotas, atribui-se a cada linha um custo inversamente proporcional à sua largura de banda

C[i][j] = 2^20 / B[i][j].

(Ou seja, o custo é calculado como o tempo gasto para se transmitir 1Gb pela linha). A rota ótima entre S e T é definida como o caminho S~~>DR~~>T de menor custo total.

IMPORTANTE:
Caso existam dois caminhos mínimos de entre A e B, deve-se escolher o menor caminho lexicograficamente da direita para esquerda. Por exemplo, se os caminhos 2-7-5-3 e 2-1-4-5-3 têm o mesmo custo total, o segundo deve ser escolhido pois, da direita para a esquerda, 3=3, 5=5, mas 4<7. Ou seja, o desempate é feito do destino para a origem pelo menor precursor.

Formato da Entrada

A entrada é constituída de K casos, cada caso correspondendo a uma configuração da rede com N nós, seguida da especificação de M mensagens. Portanto, cada caso tem a forma

N DR
D[0] R[0][0] B[0][0] R[0][1] B[0][1] ... R[0][D[0]] B[0][D[0]-1]
D[1] R[1][0] B[1][0] R[1][1] B[1][1] ... R[1][D[1]] B[1][D[1]-1]
...

D[N-1] R[N-1][0] B[N-1][0] R[N-1][1] B[N-1][1] ... R[N-1][D[N-1]] B[N-1][D[N-1]-1]
M
S[0] T[0] Z[0]
S[1] T[1] Z[1]
...

S[M-1] T[M-1] Z[M-1]
// linha em branco

onde

N = número de nós da rede (0 < N <= 2^8)
DR = roteador designado
D[i] = número de nós adjacentes ao nó i (0 < D[i] < N)
R[i][j] B[i][j] = especifica uma linha entre o nó i e o nó R[i][j] com largura de banda B[i][j]
M = quantidade de mensagens especificadas (0 < M < 2^10)
S[i] = emissor da i-ésima mensagem  (0 <= i <= M-1)
T[i] = receptor da i-ésima mensagem  (0<= i <= M-1)
Z[i] = tamanho da i-ésima mensagem em bits (0 < Z[i] < 2^31; 0 <= i <=M-1)

Formato da Saída

Para cada caso C=0,...,K-1 deve ser impressa a saída correspondente na forma

caso C:
H[0] TT[0]
H[1] TT[1]
...
H[M-1] TT[M-1]
// linha em branco

onde

H[i] = número de nós intermédios pelos quais a mensagem passa na sua rota do emissor até o receptor (excluindo emissor e receptor)
TT[i] = estimativa tempo máximo de transmissão da mensagem.

O tempo máximo de transmissão da mensagem é estimado como a soma dos tempos de transmissão da mensagem entre os nós adjacentes. Por exemplo, suponha que uma mensagem de 205 Kb deva ser transmitida de S a DR por um canal de 10Kbps e depois de DR a T por um canal de 20Kbps.

S <===== 10 Kbps ======> DR <===== 20 Kbps ======> T

No primeiro trecho, a mensagem precisará de 205/10=20s (repare na divisão inteira!) para ir de S a DR. No segundo trecho, a mensagem precisará de 205/20=10s para ir de DR a T. Portanto, o tempo total será estimado como 20s+10s = 30s usando apenas um nó intermédio (DR).
