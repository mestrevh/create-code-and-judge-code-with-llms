Descrição do Problema (problem.txt)

Título: Binary Search File System Topicos do problema: Árvore; Tempo limite de execução: 1 segundo(s) Descrição:

Zé Aluno do CIn trocou o HD do seu computador por um SSD, com o objetivo de deixar a leitura e escrita de arquivos no seu computador mais rápido. Após fazer a mudança ele percebeu que o aumento na velocidade não foi tão bom quanto ele estava esperando, e descobriu que o culpado disso era seu sistema de arquivos.

Então ele decidiu fazer um sistema de arquivos ele mesmo, mas antes de começar o trabalho ele decidiu testar sua solução através de uma simulação. Seu sistema de arquivos utilizava árvores de busca binárias para armazenar seus arquivos e diretórios, fazendo com que o acesso fosse feito em tempo logarítmico.

Nesse novo sistema de arquivos, cada arquivo contem apenas a informação do seu nome. E cada diretório, além do seu nome, possui uma BST para armazenar os arquivos e diretórios nele contidos. A raiz do seu sistema de arquivos é um diretório como qualquer outro, e como tal pode conter ter arquivos e outros diretórios. Todos os arquivos e diretórios do seu sistema possuem um nome, esse nome é único dentro do mesmo diretório.

Para simular seu sistema de arquivos, Zé Aluno do CIn vai precisar de alguns comandos básicos similares aos utilizados em bash:

  • cd D
    • Entra no diretório D, presente no diretório atual.
    • Se D for .., muda para o diretório superior.
    • Se D for .. e se o diretório atual for o diretório raiz, o comando não tem ação.
    • Caso D não exista, imprime No such file or directory.
    • Case D seja um arquivo, imprime Not a directory.
  • ls
    • Lista arquivos e pastas do diretório atual, um por linha, e em órdem lexicográfica.
    • Não imprime nada caso o diretório esteja vazio.
  • touch F
    • Cria arquivo F no diretório atual.
    • Caso F já exista, imprime File exists.
  • mkdir D
    • Cria diretório D no diretório atual.
    • Caso D já exista, imprime File exists.
  • pwd
    • Imprime o nome do diretório atual.
    • Caso o diretório atual seja o diretório raiz, imprime /.
  • tree ARG
    • Imprime recursivamente todo o conteudo do diretório atual.
    • Caso ARG seja --pre-order, os nós da árvore do diretório atual e todas as subárvores serão impressos em pré órdem.
    • Caso ARG seja --in-order, os nós da árvore do diretório atual e todas as subárvores serão impressos em órdem.
    • Caso ARG seja --post-order, os nós da árvore do diretório atual e todas as subárvores serão impressos em pós órdem.
    • Qualquer outro valor para ARG, imprimir Illegal option.
    • Se o nó a ser impresso for um arquivo, apenas imprima seu nome.
    • Se o nó a ser impresso for um diretório, imprima seu nome e logo em seguida execute a mesma função tree ARGpara ele, aumentando o nível de identação em 4 espaços.
  • rm F
    • Deleta o arquivo ou diretório F do diretório atual.
    • Caso F seja um diretório, todo seu conteudo também é deletado.
    • Caso F não exista, imprime No such file or directory.
Formato da entrada:

Cada linha da entrada contem um único comando:

  • cd D
  • ls
  • touch F
  • mkdir D
  • pwd
  • tree ARG
  • rm F

Caso qualquer outro comando seja digitado, ele deve ser considerado como inválido.

  Formato da saída:

Para cada comando válido, seguir as especificações anteriormente citadas.

Se o comando for inválido, imprimir Command not found.

Gerado automaticamente.
