Descrição do Problema (problem.txt)

Título: Duas Extremidades Topicos do problema: multiplas entradas; maratona; requer solução eficiente; recursão; programação dinâmica; Tempo limite de execução: 1 segundo(s) Descrição:

No jogo para dois jogadores "Duas Extremidades", um número par de cartas é colocado em uma linha. Em cada carta virada para cima está escrito um inteiro positivo. Jogadores jogam em turnos , em cada turno um jogador pode remover uma carta de uma das extremidades da fila, colocando a carta em sua pilha de cartas. O jogador que possuir a maior soma (somando o valor de suas cartas na pilha) vence.

Uma das Estratégias é simplismente escolher a maior carta entre as duas extremidades, podemos chamar essa estratégia de gulosa. No entanto ela nem sempre é a melhor estratégia, exemplo:

O primeiro jogador vence se escolher a carta 3 no lugar da 4 na seguinte sequência : 3 2 10 4.

Você deve determinar a maior diferença entre a pontuação dos jogadores se o segundo SEMPRE usar a estratégia gulosa, enquanto o primeiro pode usar ou não.

Formato da entrada:

A entrada consiste em vários casos de teste. Cada caso de teste contém apenas uma linha. Cada linha começa com um número par n indicando a quantidade de cartas, seguido de n inteiros positivos. O programa encerra quando n = 0.

Você pode assumir que 0 >= n <=1000(n é par), e que a soma máxima dos números da lista não ultrapassa 1,000,000.

Formato da saída:

Para cada caso de teste imprima:

In game m, the greedy strategy might lose by as many as p points.

onde m é o número do jogo (começando de 1) e p é o número da diferença máxima possível entre a pontuação do primeiro e do segundo jogador considerando que o segundo jogador sempre usa a estratégia gulosa, ou seja, sempre escolhendo a maior extremidade.

OBS: se ouver empate entre as extremidades escolha a da esquerda.

Gerado automaticamente.
