Descrição do Problema (problem.txt)

Título: Justificar Topicos do problema: array; string; Tempo limite de execução: 1 segundo(s) Descrição:

Heitor aprendeu um algoritmo novo para justificar um texto, e quer comparar o desempenho do seu algoritmo comparado com os algoritmos convencionais. Então ele te pediu para implementar o algoritmos simples de justificação de texto.

O algoritmo convencional funciona da seguinte maneira:

  1. Você deve fazer com que cada linha contenha exatamente C colunas;

  2. Para cada linha você deve fazer com que ela tenha o máximo de palavras possível, pulando para a próximo somente quando a linha atual não pode conter a palavra atual;

  3. Para uma linha (com exceção da última) os espaços adicionais devem ser adicionados entre as palavras, nunca no começo ou no final. E estes espaços devem ser igualmente distribuídos, colocando as “sobras” nas posições mais à esquerda.
    Ex: Se devo distribuir 5 espaços para 3 posições, a primeira e a segunda posição ficam com 2 espaços e a terceira com apenas 1.
    Se as linhas devem conter 24 colunas, e ela contém as palavras: “Eu”, “amo”, “muito” e “programar”; Temos um caso semelhante ao descrito acima, e resposta seria esta:
    “Eu  amo  muito programar”

  4. Deve conter pelo menos um espaço em branco entre palavras na mesma linha, e é garantido que C e as palavras são dados de tal forma que sempre podemos escrever no mínimos duas palavras em cada linha (exceto a última).

  5. A última linha conta somente com as palavras e um espaço entre elas, nada mais.
Formato da entrada:

Na primeira linha temos P e C ( 1 <= P, C <= 10000), a quantidade de palavras que serão fornecidas e a quantidade de colunas que cada linha deve ter após a justificação.

Em seguida temos todas as P palavras em uma única linha, separadas por espaço. De forma que para uma palavra Pi, |Pi| <= C / 2 - 1. Formato da saída:

O texto formatado com C colunas(com exceção da última, possivelmente) feitos da maneira descrita acima.

Gerado automaticamente.
