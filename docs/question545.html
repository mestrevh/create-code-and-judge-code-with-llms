Descrição do Problema (problem.txt)

Título: Exercitando BSTs Topicos do problema: Árvore; busca binária; Tempo limite de execução: 1 segundo(s) Descrição:

O objetivo deste exercício é implementar uma árvore de busca binária (BST) com as seguintes operações elementares:

1) ADD k : insere o elemento k na árvore caso ele não exista. Retorna 1 caso o elemento tenha sido adicionado e 0 caso ele já exista na árvore.

2) DEL k : remove o elemento k da árvore caso ele exista. Retorna 1 caso o elemento tenha sido removido e 0 se ele não existia na árvore.

3) INT a b : retorna a quantidade de elementos da árvore com valores no intervalo [a,b] (inclui as duas extremidades).

4) SGE : retorna o valor do segundo maior elemento da árvore.


Neste exercício, antes de efetuarmos as operações acima, devemos iniciar uma BST originalmente vazia conforme o procedimento a seguir.

i. Inicialmente, preenchemos dois arrays, Y e X, ambos de comprimento N, com valores pseudo-aleatórios da seguinte forma:

Y[0] = seedY
Y[i] = (Y[i - 1] * z + w) % modY, para 1 <= i < N

e

X[0] = seedX
X[i] = X[i - 1] + 1 + (Y[i - 1] % modX), para 1 <= i < N

onde seedY, z, w, modY, seedX e modX são valores dados.

ii. A adição dos elementos inciais à BST T, faz-se através da chamada à função

bst_init(T, X, 0, N-1);

onde X é o array iniciado no passo anterior e bst_init é o procedimento recursivo definido como

Função bst_init
Entrada:  T, a BST
          X=(X[0],...,X[N-1]), array de inteiros
          a, b, dois inteiros positivos com a<=b
início
  c = chão((a+b)/2)
  bst_add(T, X[c]) // adiciona X[c] à BST T cf. operação ADD
  se a <= c-1 então
    bst_init(T, a, c-1);
  fim-se
  se c+1 <= b então
    bst_init(T, c+1, b)
  fim-se
fim

Formato da entrada:

A entrada é constituída por um único caso composto de várias linhas.
A primeira linha contém 7 inteiros separados por espaços

N seedY z w modY seedX modX

onde

N é a quantidade de elementos dos arrays Y e X;
seedY, z, w, modY, seedX e modX são os parâmetros de iniciação discutidos anteriormente.

Cada uma das seguintes linhas possui um comando, correspondente a uma das operações discutidas acima,  com seus possíveis argumentos. As formas possíveis são:

ADD k
DEL k
INT a b
SGE
 
Limites:
1 <= N <= 10^6
-2^31 <= a, b, seedX, z, w, seedY, modX, modY < 2^31
-2^31 <= Y[i - 1] * z + w < 2^31, para i = 1..N
-2^31 <= X[i - 1] + 1 + (Y[i - 1] % modX) < 2^31, para i = 1..N
A quantidade de comandos não será maior que 10^4

ATENÇÃO:
Antes de qualquer operação, a BST deve ser iniciada como especificado acima.

Formato da saída:

Para cada comando na entrada, deve ser impressa uma linha com o resultado da operação correspondente.

ADD b: imprime 1 se o elemento foi inserido, ou 0, se o elemento já existia.

DEL b: imprime 1 se o elemento foi removido, ou 0, se o elemento não existia.

INT a b: imprime a quantidade de elementos no intervalo [a,b].

SGE: imprime o valor do segundo maior elemento da árvore ou -1 caso exista menos de dois elementos.

ATENÇÃO:
As inserções iniciais feitas pela função bst_init NÃO devem ter os seus retornos impressos.
