Descrição do Problema (problem.txt)

Título: Partição do espaço Topicos do problema: Árvore; busca binária; Tempo limite de execução: 3 segundo(s) Descrição:

Em computação gráfica, a técnica de partição recursiva do espaço é utilizada para desenhar cenas com diversos objetos, evitando sobreposições indesejadas. Uma versão simplificada dessa técnica é descrita a seguir.

Suponha que uma cena possua N objetos, numerados de 0 a N-1, cada um à sua respectiva distância D[0], D[1], ..., D[N-1] da câmera (ponto de vista do observador). Gostaríamos de desenhá-los em ordem decrescente de distância, ou seja, dos mais distantes aos mais próximos, de forma que a imagem de um objeto mais distante nunca se sobreponha a de um mais próximo.

A técnica consiste em supor que, inicialmente, todos os objetos pertencem simultaneamente a um mesmo conjunto. Escolhe-se então um objeto P nesse conjunto {0, ..., N-1}. Esse elemento é usado como base para particionar os restantes objetos em dois subconjuntos: o subconjunto S' = {i | D[i] < D[p]} e o conjunto S" = {i | D[i] >= D[P]}. O processo é então repetido recursivamente em S' e S" caso eles não sejam vazios. Repare que esse conjunto, assim particionado, pode ser representado por uma BST na qual a raiz é o objeto P e as subárvores à esquerda e à direita correspondem aos conjuntos S' e S", respectivamente.

Adicionalmente a essa classificação inicial, podemos supor um cenário dinâmico, por exemplo, um jogo, no qual objetos são constantemente adicionados e removidos da cena. Temos então as seguintes operações:

  1. ADD D: adicionar um objeto a uma certa distância D da câmera à cena
  2. DEL L R: remover todos os objetos da cena cuja distância à câmera situa-se no intervalo [L, R]
Formato da entrada:

Neste exercício, a entrada consiste, inicialmente, da especificação de uma cena contendo N objetos a distâncias D[0], ..., D[N-1].

Entretanto, ao invés de ser dada explicitamente, essa sequência é especificada como uma sequência pseudo-aleatória obtida através de um gerador congruente linear da seguinte maneira:

  • D[0] = seedD, onde 0 < seedD < M é um número dado chamado de "semente" da sequência
  • D[i] = (A * D[i-1] + C) % M, para 1 <= i < N, sendo A e C constantes dadas.

A primeira linha da entrada contém, portanto, 5 inteiros N M seedD A C, sendo:

  • N = quantidade inicial de objetos na cena
  • M = limite superior do valor das distâncias
  • seedD = semente do gerador de distâncias
  • A, C = parâmetros do gerador discutidos acima

A segunda linha contem um inteiro Q, indicando a quantidade de operações de adição/remoção de objetos a serem executadas.

Seguem-se Q linhas, cada uma com um comando correspondente a uma das operações discutidas acima, com seus possíveis argumentos. As formas possíveis são:

  • ADD D
  • DEL L R

Limites:

1 <= N <= 5*10^5
0 < seedD < M <= 10^9
1 <= A, C <= 10^9
1 <= Q <= 10^5
-10^9 <= K <= 10^9
-10^9 <= L <= R <= 10^9

ATENÇÃO:

  • Use inteiros de 64 bits (int64) para efetuar as operações do gerador
  • As chaves obtidas através do gerador serão sempre distintas
Formato da saída:

Inicialmente, as N distâncias especificadas pela sequência aleatória devem ser particionadas como descrito acima. O objeto P de (0, ..., N-1), usado como pivô da partição, deve ser escolhido com base na mediana entre D[0], D[(N-1)/2] e D[N-1]. Por exemplo, caso tivéssemos a sequência de distância D=(7, 3, 8, 4, 2, 1, 5) teríamos D[0]=7, D[3=6/2]=4, e D[6]=5, e portanto, o objeto correspondente à distância mediana seria P=6 (posição à extrema direita). Essa mesma regra deve ser aplicada a todo momento em qualquer subsequência. Se, entretanto, a subsequência tiver menos de 3 elementos, deve ser escolhido o primeiro objeto, i.e. P=0.

A primeira linha da saída deverá ser da forma:

0: H

onde H é um inteiro correspondente à altura inicial da árvore de particionamento.

Em seguida, para cada uma das Q operações j=1, ..., Q, imprimir uma linha com o resultado da operação como a seguir:

1. Se a operação for ADD D, imprimir:
j: A

onde:

  • j é o número da operação
  • A = 0 se a cena já possui um objeto à distância D, caso no qual a operação resulta em falha e o objeto não deve ser adicionado. na adição do valor D à cena
  • A = 1 caso a cena não possua um objeto à distância D, caso no qual a operação deve adicioná-lo
2. Se a operação for DEL L R, imprimir:
j: K

onde:

  • j é o número da operação
  • K >= 0 é o número de objetos removidos da cena
