Descrição do Problema (problem.txt)

Título: Bitmap II Topicos do problema: dividir e conquistar; Tempo limite de execução: 1 segundo(s) Descrição:

O bitmap é uma estrutura de dados que é utilizada em muitas áreas da computação. Na Computação Gráfica, por exemplo, um bitmap pode representar uma imagem, onde 1 seria um pixel preto e 0 um pixel branco.

Mas as formas de representar um bitmap podem ser bem variadas. Neste problema, o objetivo é transformar de uma forma para outra.

A forma inicial é uma string composta de 0s, 1s e Ds. A forma final é a representação dessa string num array bidimensional (vamos chamar de B).

Inicialmente, a matriz B toda é considerada. Se na string houver um D, é necessário dividir B em 4 quadrantes, que deverão ser considerados na seguinte ordem: superior esquerdo, superior direito, inferior esquerdo e inferior direito. Se na string houver um 1, toda a parte atual de B deve ser preenchida com 1s, e o mesmo serve para o 0. Dessa forma, será possível converter o bitmap de sua compacta forma de string para o bitmap original.

Quando um bitmap possui um número par de linhas e também de colunas, então todos os quadrantes possuirão as mesmas dimensões.

Quando o número de colunas é ímpar, os quadrantes da esquerda devem ter uma coluna a mais do que os quadrantes da direita.

Quando o número de linhas é ímpar, os quadrantes superiores devem possuir uma linha a mais do que os quadrantes inferiores.

Seu objetivo é ler a o bitmap na sua forma de string compacta e transformá-la na forma de matriz.

Formato da entrada:

Existirão múltiplas entradas, em cada uma delas:

Será dado um inteiro n e outro m, representando respectivamente a quantidade de linhas e colunas da forma matricial do bitmap. (1 <= n, m <= 200)
Logo em seguida, será dada uma string s com a representação compacta do bitmap B.

Formato da saída:

A saída consistirá dos bitmaps na sua forma matricial. Separe cada caso de teste por uma linha em branco.

Gerado automaticamente.
