Descrição do Problema (problem.txt)

Título: Loucos por compras Topicos do problema: backtracking; Tempo limite de execução: 3 segundo(s) Descrição:

A vencedora de um concurso de um programa de televisão recebeu como prêmio a oportunidade de gastar uma avultada quantia com uma lista de itens de uma seleção de produtos numa das lojas de uma rede de hipermercados. No entanto, as regras do concurso impõem que, no máximo, um item de cada produto e, no total, pelo menos M itens sejam selecionados.
 
Para fazer o melhor uso possível do prêmio, a felizarda ganhadora decidiu avaliar de quantas maneiras ela poderia escolher os itens utilizando o valor do prêmio e lançou-se à laboriosa tarefa de percorrer as lojas para coletar os preços de todos os produtos à disposição.

Formato da entrada:

A entrada é constituída por vários casos correspondendo ao levantamento de várias lojas.

A primeira linha da entrada contém um inteiro K correspondendo ao número de casos, seguida por uma linha em branco.

Em seguida, temos os K casos, cada caso no seguinte formato:

L
P[0] P[1] ... P[L-1]
M Q
//linha em branco

onde

L = número de itens disponíveis 0<L<31
P[j] = preço do item j
M = menor quantidade de itens que se pode escolher
Q = quantia máxima que pode ser gasta


Formato da saída:

Para cada caso C=0,...,K-1 deve ser impressa uma linha na forma

caso C: R

onde

R = número total de possíveis maneiras de formar uma compra no valor máximo Q do prêmio, escolhendo-se, no mínimo, M itens sendo, no máximo, uma unidade de cada tipo, dentre os itens disponíveis.
