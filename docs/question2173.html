Descrição do Problema (problem.txt)

Título: P3Q2 - Cadastro de partidas Topicos do problema: estrutura de dados; acumulador; Tempo limite de execução: 1 segundo(s) Descrição:

Os juízes da corrida subterrânea gostariam de melhorar o sistema que utilizam, criando um repositório, que deve ser implementado com Listas Encadeadas, permitindo cadastrar o nome da toupeira toda vez que ela competir. Assim, o nome de uma toupeira pode aparecer diversas vezes na base de dados. O objetivo é descobrir quantas vezes uma toupeira participou da competição.

Formato da entrada:

A primeira linha contém um inteiro N com o número de partidas ocorridas no dia.


A segunda linha contém N strings com o nome de uma toupeira, indicando que ela está competindo nessa partida.


A terceira linha é composta de uma única string, referente ao nome da toupeira que deverá ser consultada.


Formato da saída:

A saída é composta por um único inteiro, representando a quantidade de vezes que a toupeira informada participou da competição.
