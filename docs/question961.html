Descrição do Problema (problem.txt)

Título: Piscamos tanto assim? Topicos do problema: array; alocação dinâmica; Tempo limite de execução: 3 segundo(s) Descrição:

Antonio Cabral Monteiro percebeu que piscamos os olhos muitas vezes em um minuto. Então, decidiu observar por vários dias quantas vezes sua vizinha Ana Gabriela Orsi piscava os olhos. Sabendo que você é um sábio programador, ele pediu sua ajuda para fazer um banco de dados, e,  com suas informações, estudar os números que ele obtém todos os dias.

Antonio quer que, no seu banco de dados, seja possível:

  1. Inserir informações coletadas em um novo dia

  2. Deletar as informações de um dia (as vezes ele percebe que foi um dia estranho).

  3. Saber qual o maior número de piscadas em um minuto no i-ésimo dia.

  4. Saber o maior número de piscadas no j-ésimo minuto de observação de todos os dias (se j for maior que o tempo de observação de algum dia, considere que a quantidade de piscadas desse minuto neste dia é 0).


OBS:É garantido que:

- A soma da quantidade de dados armazenados por Antônio é menor que 10⁹;

- Podemos ter no máximo 10⁶ dias distintos salvos;

- O máximo de minutos por dia é 1140 (logicamente);

- Os dias e os minutos de cada um destes vão começar em 0.



Formato da entrada:

No começo de cada linha da entrada temos um inteiro t, identificando qual o tipo de consulta Antonio deseja fazer, e seu programa deve executar até que t seja igual a 0.

Caso t == 1:

    Inserir um novo dia - Você receberá um inteiro Q(1 <= Q <= 1140), representando a quantidade de minutos avaliados no novo dia, seguido de Q inteiros (0 <= i < Q, 0 <= Pi <= 1000 ), onde cada Pi é o número de piscadas no i-ésimo minuto. Esse novo dia deve ser adicionado na linha n+1, para n linhas já ocupadas.

Caso t == 2:

   Deletar um dia - Você receberá um inteiro d, indicando o dia que deverá ser deletado do banco de dados, sendo necessária a aplicação das devidas operações para que se mantenha a ordem da tabela. Ou seja, o ideal é que, após a remoção, a matriz continue contínua assim como era.

Caso t == 3:

    O maior número de piscadas registrado em determinado dia - Você receberá um inteiro i, indicando o índice do dia que deve ser avaliado. Assuma que i é um número válido, ou seja, 0 <= i < n, onde n é o número de dias salvos atualmente no banco de dados.

Caso t == 4:

    O maior número de piscadas registrado em determinado minuto - Você receberá um inteiro j (0 <= j < 1140), indicando o índice do minuto que será avaliado.

Formato da saída:

Para cada linha de entrada, mostra um inteiro indicando:

Caso t ==  1:

    O número total de dias salvos no banco de dados após a adição deste último.

Caso t == 2:

    Quantos minutos foram removidos no d-ésimo dia deletado.

Caso t == 3:

    O maior número de piscada no i-ésimo dia.

Caso t == 4:

    O maior número de piscadas no j-ésimo minuto de observação de todos os dias.

Gerado automaticamente.
