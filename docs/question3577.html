Descrição do Problema (problem.txt)

Título: O Robô do Zeca Topicos do problema: string; Tempo limite de execução: 1 segundo(s) Descrição:


Zeca fez um robô movido à armas de nerf, existem quatro arminhas no robô, vendo de cima ele tem uma arminha para cada uma das quatro direções cima, baixo, esquerda, direita.

Se o robô atirar uma arminha, ele vai se mover para a direção oposta, por exemplo: Se o Robô está na posição (0, 0) e atirar para esquerda e depois para baixo, ele irá para (1, 0) no primeiro tiro e (1, 1) depois do segundo.

O robô irá iniciar na posição (0, 0) e zeca quer que ele chegue na posição (x, y). Zeca fez um programa para o robô se movimentar de forma que gastasse a munição de todas as arminhas, porém ele percebeu que nem todo programa que ele fez faz o Robô chegar no objetivo (x,y) e agora ele precisa de sua ajuda.

Você receberá o programa que Zeca programou para o robô e deverá dizer para Zeca se é possível remover algumas intruções de atirar (sem mudar as ordens das instruções) de forma que ele chegue no objetivo.


O programa CCCEBED significa que o robô irá atirar a arma de cima três vezes, depois a arma da esquerda, etc...

Instruções:

C: o robô na posição (a, b) irá para (a, b-1)

B o robô na posição (a, b) irá para (a, b+1)

E: o robô na posição (a, b) irá para (a+1, b)

D o robô na posição (a, b) irá para (a-1, b)

Formato da entrada:

Na primeira linha o objetivo do robô: x y

Na segunda linha o tamanho da string com as instruções: n

Na terceira linha, uma string com as n instruções: str

Formato da saída:

S, caso seja possível remover instruçoes do programa do Zeca para o seu Robô chegar no objetivo e N caso seja impossível.

Gerado automaticamente.
