Tópicos do problema

recursão

Devorador de Mentes

Tempo limite: 1 segundo(s)

ODe volta ao mundo normal, o Devorador de Mentes decide atacar Hawkins novamente e escravizar os habitantes. Will e seus amigos, percebendo que a maioria dos cidadãos estavam com um comportamento estranho, decidiram investigar quão forte o Devorador de Mentes já estava.



Eles sabiam que era uma tarefa um pouco difícil e decidem pedir sua ajuda para conseguir saber detalhadamente a força de todos os súditos.


Através de informações sobre a força de cada cidadão possuído pelo Devorador de Mentes você deve construir uma função recursiva de soma que mostre o poder atual do monstro. Este poderia ser calculado através da soma de todos os membros que possuem força maior que 10. Os demais poderiam ser desprezados no cálculo. Para tornar o cálculo mais rápido, você precisa: 

  1. particionar cada grupo, representado por um array, em 2 subgrupos; 

  2. aplicar a função recursiva de soma na metade esquerda do array;

  3. aplicar a função recursiva de soma na metade direita do array;

  4. somar o resultado retornado pela aplicação da função recursiva às duas partes do array (metade direita e metade esquerda);

  5. retornar a soma do passo (4) como resultado da função.


Obs: caso a quantidade de elementos no array seja ímpar, na hora de particionar o array, você decide se coloca um elemento a mais do lado direito ou do lado esquerdo.

Obs: A resolução da questão deve ser feita usando função recursiva.


A função recursiva termina quando houver apenas um elemento no array, tornando impossível o particionamento. Quando atingir esse ponto, o valor unitário será retornado como resposta, caso ele seja maior do que 10.


Vamos ver um exemplo:


Imaginemos a função soma do array representada por S(a).

Imagine o array = {21, 45, 2, 11}

  • Sua soma é dada por: S({21, 45}) + S({2, 11}).

  • A soma do array {21, 45} pode ser expressa por: S({21}) + S({45})

  • A soma do array {21}, que só possui um elemento, será 21. Logo: S({21}) = 21

  • A mesma ideia é válida para a soma do array {45}. Logo S({45}) = 45

  • Por sua vez a soma do array {2, 11} pode ser expressa por S({2}) + S({11})

  • A soma do array {2}, como ele só tem um elemento menor que 10 é 0. Logo: S({2}) = 0

  • A soma do array {11}, que só possui um elemento, será 11. Logo S({11}) = 11

  • Voltando os passos, temos que S({21, 45, 2, 11}) = 21 + 45 + 0 + 11 = 77.

(essa é uma possível solução que particiona o problema e não percorre o array linearmente)


Vamos agora ver um exemplo de um array com número ímpar de elementos.


Imaginemos a função soma do array representada por S(a).

Imagine o array = {21, 2, 11}

  • Sua soma pode ser dada por: i) S({21}) + S({2, 11}) ou por ii) S({21, 2}) + S({11}). Vamos escolher a opção (i).

  • A soma do array {21}, que só possui um elemento, será 21. Logo: S({21}) = 21

  • Por sua vez a soma do array {2, 11} pode ser expressa por S({2}) + S({11})

  • A soma do array {2}, como ele só tem um elemento menor que 10 é 0. Logo: S({2}) = 0

  • A soma do array {11}, que só possui um elemento, será 11. Logo S({11}) = 11

  • Voltando os passos, temos que S({21, 2, 11}) = 21 + 0 + 11 = 32.

(essa é uma possível solução que particiona o problema e não percorre o array linearmente)

Formato da Entrada

N

X1 X2 X3… Xn


Sendo N um número natural representando a quantidade de membros total e Xi a respectiva força que o i-ésimo membro possui.


Formato da Saída

S

Sendo S um número natural representando a soma da força calculada para os indivíduos utilizando a função descrita.
