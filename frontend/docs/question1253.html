Tópicos do problema

busca

Caça-penetras

Tempo limite: 2 segundo(s)

Será realizada no CIn uma festa de confraternização. Para ter acesso à festa, cada convidado receberá uma senha aleatória única com valores num certo intervalo 0..K-1. Visando evitar penetras, os três monitores foram informados que participarão da organização de um sistema que identificará se a senha dada está ou não cadastrada, e essa pesquisa deve ser feita o mais mais rápido possível. Todos concordaram em usar hashtables de tamanho M, mas eles não chegaram a um acordo quanto aos detalhes da implementação.

  • O monitor A defende que o melhor seria um hash aberto com vetores dinâmicos e inserções sempre no final.

  • O monitor B afirma que o melhor metódo seria utilizando um hash aberto com vetores dinâmicos, mas com inserções de forma que os elementos estivessem sempre ordenados, de forma que as buscas possam ser realizadas em tempo logarítmico.

  • O monitor C diz que o melhor seria fazer um hash fechado usando um hash duplo pra resolver colisões, com funções de hashing h0(k) = k mod M e h1(k) = (2 * (k mod (M / 2))) + 1.

Em qualquer uma das alternativas, todos concordaram em que, caso o fator de carga L torne-se >= 0.75, deve ser realizado um rehash da tabela, atualizando o tamanho para 2 * M, e recolocando todos os elementos da tabela da primeira à última posição, e na ordem em que aparecem nas listas de cada posição.

Para resolver a dúvida de qual sistema de busca seria o mais eficiente, você precisa contar quantas comparações são necessárias para encontrar as senhas nos diversos cenários.

Formato da Entrada

A entrada é constituída de T casos de teste, numerados 0..T-1.

Em cada caso, temos uma primeira linha com quatro inteiros

K M N Q

onde

  • K indica quantidade de possíveis senhas, numeradas 0..K-1.
  • M é o tamanho da tabela hash inicial
  • N indica a quantidade de senhas a serem inseridas nas tabelas.
  • Q indica a quantidade de consultas a serem realizadas.

Em seguida, temos N linhas

s[0]
s[1]
...
s[N-1]

onde cada s[i] é uma senha distinta a ser inserida em cada uma das três tabelas.

Após isso, temos Q linhas

v[0]
v[1]
...
v[Q-1]

onde cada v[j] é uma senha distinta que deve ser procurada em cada uma das tabelas previamente preenchidas.

Após cada caso, temos uma linha em branco.

Formato da Saída

Para cada caso t=0..T-1, as N senhas s[0..N-1] devem ser inseridas nas tabelas A, B e C, inicialmente vazias, conforme descrito acima. Em seguida, cada uma das Q senhas v[0..Q-1] devem ser procuradas nas três tabelas. Ao final das Q consultas, o programa deve imprimir uma linha na forma

caso t: P[t] A[t] B[t] C[t]

onde

  • t indica o número do caso
  • P[t] indica o número de "penetras", ou seja, quantas das Q senhas pesquisadas não estavam nas tabelas
  • A[t] B[t] C[t] indicam os números totais de comparações feitas nas Q consultas nas tabelas A, B e C, respectivamente.
