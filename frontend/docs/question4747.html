Nível de Programador

Tempo limite: 1 segundo(s)

Onaibaf, o gerente da lendária startup BugLess Corp, está cansado de entrevistar candidatos que dizem ser "ninjas da programação", mas travam no primeiro scanf. Para resolver esse problema, ele pediu sua ajuda para desenvolver um sistema automatizado que avalie o nível real de experiência dos candidatos, com base em dados mais objetivos.

Seu programa será usado nas entrevistas técnicas e deve receber os seguintes dados do candidato:

  1. Quantidade de linhas de código escritas na vida (inteiro positivo)

  2. Quantidade de xícaras de café consumidas durante programação (inteiro positivo)

  3. Quantidade de bugs causados que ele mesmo admitiu (inteiro positivo)

O programa deve verificar se algum dos valores é zero e, caso seja, exibir uma mensagem específica e encerrar a execução (sem calcular o índice).

  • Se o número de linhas de código for 0:
Você nunca escreveu uma linha de código? Vá compilar coragem primeiro!
  • Se a quantidade de café for 0:
Sem café? Programação sem cafeína é só digitação triste.
  • Se a quantidade de bugs for 0:
Nenhum bug? Ou você é um gênio ou está negando a realidade.

Importante: apenas uma dessas mensagens deve ser exibida — verifique na ordem: linhas → café → bugs.

O programa então calcula um índice de programação, segundo a fórmula:

índice = (linhas_de_codigo / 1000.0) + (cafe * 2) - (bugs / 5.0)

Com esse índice, classifique o candidato de acordo com a seguinte escala:

  • Índice < 10
Nível: Estagiário curioso. Ainda não causou grandes bugs (tirando os que ele mesmo causou).
  • Índice >= 10 e < 20
Nível: Dev júnior. Já sabe que 'segmentation fault' não é um elogio.
  • Índice >= 20 e < 40
Nível: Dev pleno. Domina o CTRL+C, o Stack Overflow e o café sem açúcar.
  • Índice >= 40 e < 80
Nível: Dev sênior. Café no sangue, código na veia, e bug nos outros sistemas.
  • Índice >= 80
Nível: Lenda da programação. Há boatos de que ele já usou ponteiros com os olhos fechados.

Formato da Entrada

O programa espera que você informe:

  1. A quantidade total de linhas de código que já escreveu;
  2. Quantas xícaras de café você já tomou programando;
  3. Quantos bugs você mesmo causou (e admitiu).

Exemplo:

20000
10
15

Formato da Saída

A saída deve conter:

  1. O índice de programação, com uma casa decimal.

  2. Logo após, o nível do programador, exatamente como descrito na tabela de classificações.


Exemplo:

Índice de programação: 37.0
Nível: Dev pleno. Domina o CTRL+C, o Stack Overflow e o café sem açúcar.
