O Tribunal de Contas da União contratou você para desenvolver um sistema capaz de detectar fraudes nas contas públicas. O sistema receberá uma série de dados da entrada padrão e analisará se há fraude em potencial e qual o tipo de fraude pode ocorrer.

  • O sistema receberá primeiro qual o tipo de despesa:
    • Pagamento para pessoa física (PPF)
    • Pagamento para pessoa jurídica (PPJ)
    • Pagamento de pessoal (PP)
  • Em seguida, o sistema receberá o valor do pagamento.
  • Por fim, se o pagamento NÃO for de pessoal (PP) o sistema deve receber qual a rubrica a que se destina, que pode ser:
    • "Capital" (CAP)
    • "Custeio" (CUST)
    • ou "Serviço" (SERV).
  • O pagamento para pessoa física só pode ser de serviço, caso contrário será emitida a mensagem "Pessoa física não pode fornecer materiais para o serviço público."
  • O pagamento para pessoa física (PPF) que ultrapassar R$ 8000,00 será marcado como "Excesso de pagamento para pessoa física.".
  • O pagamento para pessoal (PP) que ultrapassar R$ 4000,00 será marcado como "Excesso de pagamento para pessoal.".
  • Pessoa jurídica pode atender qualquer rubrica, mas os limites de valores devem ser respeitados da seguinte forma:
    • CAP: Até 5 milhões de reais
    • CUST: Até 2 milhões de reais
    • SERV: Até R$ 500.000,00
  • Se os valores ultrapassarem o permitido, o sistema deve emitir respectivamente as seguintes mensagens:
    • Para CAP: "O valor para pagamento da rubrica de capital não pode exceder R$ 5M."
    • Para CUST: "O valor para pagamento da rubrica de custeio não pode exceder R$ 2M."
    • Para SERV: "O valor para pagamento da rubrica de serviços não pode exceder R$ 500K."
  • Caso o pagamento não apresente nenhuma dessas irregularidades, o sistema deve imprimir a seguinte mensagem:
    • "Pagamento liberado."
  • Caso o usuário digite um valor inválido para o tipo de despesa, emita a mensagem:
    • "Depesa inválida."

Formato da Entrada

  • Tipo de despesa, que pode ser:
    • Pagamento para pessoa física (PPF)
    • Pagamento para pessoa jurídica (PPJ)
    • Pagamento de pessoal (PP)
  • Valor do pagamento (float)
  • Se o pagamento não for de pessoal o sistema também deve receber a rubrica, que pode ser:
    • Capital (CAP)
    • Custeio (CUST)
    • Serviço (SERV).

Formato da Saída

Uma mensagem que especifica se o pagamento contém irregularidades ou se pode ser liberado:

  • O pagamento para pessoa física só pode ser de serviço, caso contrário será emitida a mensagem "Pessoa física não pode fornecer materiais para o serviço público."
  • O pagamento para pessoa física que ultrapassar R$ 8000,00 será marcado como "Excesso de pagamento para pessoa física.".

  • Se os valores ultrapassarem o permitido, o sistema deve emitir respectivamente as seguintes mensagens:

    • Para CAP: "O valor para pagamento da rubrica de capital não pode exceder R$ 5M."
    • Para CUST: "O valor para pagamento da rubrica de custeio não pode exceder R$ 2M."
    • Para SERV: "O valor para pagamento da rubrica de serviços não pode exceder R$ 500K."
  • Caso o pagamento não apresente nenhuma dessas irregularidades, o sistema deve imprimir a seguinte mensagem:
    • "Pagamento liberado."
